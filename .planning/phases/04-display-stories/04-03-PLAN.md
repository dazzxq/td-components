  <action>
    Port dcms-table.js (TableSimple) to a Web Component extending TdBaseElement. Per D-01, data via JS property, integrated pagination (uses td-pagination internally, not as separate component).

    **Component: td-table**

    Attributes:
    - `per-page` (number, default 10)
    - `active-color` (CSS color for pagination, default '#ef4444')
    - `zebra` (boolean, default true — use booleanAttributes)
    - `loading` (boolean — use booleanAttributes)
    - `loading-rows` (number, default 5)
    - `title` (string, optional table title)
    - `empty-text` (string, default 'Khong co du lieu')
    - `server-mode` (boolean — use booleanAttributes)
    - `total-items` (number, for server mode)

    JS Properties (set via JavaScript, not attributes):
    - `columns` — array of `{key, label, sortable?, width?, widthType?, minWidth?, maxWidth?, align?, render?}`. The `render` function receives `(row, index)` and returns string (HTML), HTMLElement, or plain value.
    - `data` — array of row objects
    - `onSort` — callback `({key, direction})` for server mode
    - `onPageChange` — callback `(page)` for server mode

    Internal State:
    - `_sortState = { key: null, direction: null }`
    - `_currentPage = 1`

    Behavior (port from DCMS TableSimple):
    1. **Glass card container**: rounded-xl, bg `rgba(255,255,255,0.85)`, border, box-shadow — match DCMS exactly
    2. **Header section**: title (if set) + pagination in header. Flex between.
    3. **Table head**: bg-gray-50/60, uppercase text-xs font-semibold. Sortable columns get a button with sort indicator SVG (use inline SVG arrows, NOT Font Awesome):
       - Unsorted: both arrows, opacity-40
       - Ascending: up arrow, opacity-100
       - Descending: down arrow, opacity-100
    4. **Sort cycle**: null -> asc -> desc -> null (port `_handleSort` exactly)
    5. **Table body**: rows with hover bg transition. Zebra: odd rows get `rgba(0,0,0,0.015)` bg.
    6. **Column width**: Support widthType 'fixed' (exact width on th+td) and 'flexible' (auto with optional min/max).
    7. **Custom render**: If column has `render` function, call it. Handle string (innerHTML), HTMLElement (appendChild), other (textContent).
    8. **Empty state**: When no data, show td-empty-state (import from ./td-empty-state.js) with inbox icon, compact, size sm.
    9. **Pagination**: Render `<td-pagination>` in header and footer. Listen for `page-change` event. In client mode, slice data. In server mode, call onPageChange callback.
    10. **Loading skeleton**: When loading=true, render skeleton with animate-pulse. Column headers still shown. Body rows are gray bars with random widths (60-100%).
    11. **Footer**: Glass footer bg-gray-50/50 with pagination.

    Client-side sorting: Port `_getCurrentRows` — sort by key with null handling, numeric vs string comparison, then paginate.

    Public API methods on element:
    - `setData(data)` — update data array, reset pagination
    - `setPage(page)` — navigate to page
    - `setLoading(bool)` — toggle loading state
    - `getState()` — returns {columns, data, page, perPage, sort}
    - `update(opts)` — merge options and re-render

    Override `attributeChangedCallback`: Only re-render for meaningful attribute changes. Avoid re-render loops.

    Register: `if (!customElements.get('td-table')) customElements.define('td-table', TdTable);`
  </action>
