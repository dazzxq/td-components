    Port from dcms-dropdown.js (614 lines). Per D-04, port ALL features. Per D-13:

    **Class `TdDropdown extends TdBaseElement`**, tag `td-dropdown`:

    - observedAttributes: `['placeholder', 'searchable', 'disabled', 'allow-clear', 'max-height', 'value-key', 'label-key', 'value']`
    - booleanAttributes: `['searchable', 'disabled', 'allow-clear']`
    - Defaults: placeholder='Chon mot tuy chon', searchable=true (default on unless attr removed), allowClear=true, maxHeight=5, valueKey='value', labelKey='label'
    - Complex data (options array) set via JS property: `el.options = [{value, label}, ...]` — per D-01

    **Architecture (key difference from DCMS):**
    The DCMS Dropdown appends a menu element to `document.body` for fixed positioning. Port this pattern:
    - render() creates the button trigger inside the component element
    - afterRender() creates the menu element and appends to document.body
    - disconnectedCallback: remove menu from body, cancel RAF, remove global listeners
    - Track the menu element in `this._menuElement` for cleanup

    **Button trigger**:
    - Full width, border rounded-xl, flex with selected text + arrow SVG
    - Style: bg `rgba(255,255,255,0.72)`, border `rgba(0,0,0,0.1)`, height 40px
    - Focus: boxShadow ring blue, borderColor blue
    - Arrow rotates 180deg when open

    **Menu popup** (position: fixed, appended to body):
    - Glass styling: bg `rgba(255,255,255,0.72)`, backdropFilter `blur(16px) saturate(160%)`, border `rgba(255,255,255,0.5)`
    - z-index: 10010
    - Optional search input at top (when searchable)
    - Options list with max-height = maxHeight * 40px, overflow-y auto

    **Search filtering** (per D-13):
    - Filter options by label.toLowerCase().includes(query)
    - Reset filter on close

    **Keyboard navigation** (per D-13):
    - ArrowDown: highlight next option (wrap to first)
    - ArrowUp: highlight previous option (wrap to last)
    - Enter: select highlighted option
    - Escape: close dropdown, focus button
    - Hover: update highlight on mousemove

    **Auto-positioning** (per D-13):
    - Open below button by default (rect.bottom + 8)
    - If menu would overflow viewport bottom, open above (rect.top - menuHeight - 8)
    - If above would go negative, fallback to top: 8
    - Use position:fixed with viewport coords (no scrollX/scrollY)

    **RAF-throttled scroll** (per D-13):
    - On scroll (capture: true), throttle position updates via requestAnimationFrame
    - Cancel pending RAF on close/destroy

    **Clear selection** (per D-13):
    - When allowClear and item selected, show clear option at top: icon + "Khong chon" text
    - Clear sets selectedItem=null, shows placeholder

    **Close on outside click**: global click listener added on open, removed on close

    **Callback modes**:
    - `onChange` callback: receives value only
    - `onSelect` callback: receives full item object
    - Set via JS property: `el.onChange = (value) => {}` or `el.onSelect = (item) => {}`
    - Also emit `change` event via `this.emit('change', { value, item })` — per D-02, D-03

    **Public API methods**:
    - `getValue()` — returns selected value or null
    - `setValue(value)` — programmatic set, no callback fired
    - `getSelectedItem()` — returns full selected item object
    - `updateData(newData)` — replace options array, re-render options
    - `open()` / `close()` — programmatic open/close
    - `destroy()` — full cleanup (remove menu from body, listeners, RAF)

    **XSS safety**: use `this.escapeHtml()` for all label/value rendering in HTML

    **Static**: `TdDropdown._openDropdowns = []` for closeAllExcept pattern

    **Cleanup on disconnect**: Override disconnectedCallback, call super, remove menu from body, cancel RAF, remove global listeners.

    **Story file** — CSF format:
    - title: 'Form/Dropdown'
    - Stories: Default, WithSearch, Disabled, Preselected, ClearButton
    - Each story sets options via script after render: `setTimeout(() => { document.querySelector('td-dropdown').options = [...]; }, 0)`
