  <action>
    Port from dcms-datetime.js (FormatTime class). Per D-05, port nguyên — no date-fns/dayjs. Per D-14:

    **Class `TdDateTime`** (pure utility, no DOM dependency — export as named export):

    1. `static _parseDate(input)` — Parse ISO string, Unix timestamp (seconds < 1e12, else ms), Date object. Return Date or null.

    2. `static _pad(num, length=2)` — Pad with leading zeros.

    3. `static toAbsolute(dateInput, format = 'DD/MM/YYYY - HH:mm')` — Format tokens per D-14:
       - DD (01-31), MM (01-12), YYYY (2025), YY (25)
       - HH (00-23), hh (01-12), mm (00-59), ss (00-59)
       - A (AM/PM), a (am/pm)
       - Replace in order: YYYY, YY, MM, DD, HH, hh, mm, ss, A, a

    4. `static toRelative(dateInput)` — Vietnamese relative time per D-14:
       - < 60s: "Vua xong"
       - < 60min: "{X} phut truoc"
       - < 24h: "{X} gio truoc" (>= 30min remainder: "Hon {X} gio truoc")
       - < 30 days: "{X} ngay truoc" (>= 12h remainder: "Hon {X} ngay truoc")
       - < 12 months: "{X} thang truoc" (>= 15d remainder: "Hon {X} thang truoc")
       - else: "{X} nam truoc" (>= 6mo remainder: "Hon {X} nam truoc")
       NOTE: Use actual Vietnamese diacritics from DCMS: "Vừa xong", "phút trước", "giờ trước", "ngày trước", "tháng trước", "năm trước", "Hơn"

    5. `static convert(dateInput, options = {})` — Wrapper with `mode: 'absolute'|'relative'`, `format` for absolute.

    6. `static toISO(formattedDate, inputFormat)` — Parse formatted date back to ISO string. Build regex from format tokens, extract values, construct Date, return `.toISOString()`.

    **Tests** using Node test runner (same pattern as td-base-element.test.js):
    - Test each method with multiple inputs
    - Test edge cases: null, undefined, invalid strings, boundary values
    - Test Vietnamese output strings with correct diacritics

    - booleanAttributes: `['disabled', 'readonly', 'required']`

    **Input types** (per D-11): text, password, email, tel, number, textarea, contenteditable
    - `type` attribute determines which HTML element to create
    - textarea: `<textarea>` with rows=4 default, resize:vertical
    - contenteditable: `<div contenteditable="true">` with placeholder behavior
    - others: `<input type="...">` 

    **Sizes**: sm (h=32px, px=12, py=6, text-sm, radius=10px), md (h=40px, px=14, py=8, text-sm, radius=12px), lg (h=48px, px=16, py=10, text-base, radius=14px)

    **Character/word counter** (per D-11):
    - When `max-length` attribute is set, show counter div below input: `{current}/{max} {unit}`
    - `limit-type` attribute: 'char' (default) or 'word'
    - Unit text: 'ky tu' for char, 'tu' for word (Vietnamese: "ký tự", "từ")
    - Counter turns red (#ef4444) when at/over limit
    - Word limit: split by whitespace, truncate excess words on input
    - Char limit: use HTML5 maxLength for input/textarea, manual truncate for contenteditable

    **Label**: rendered as `<label>` above input, with red asterisk `<span class="text-red-500">*</span>` when required

    **Error/helper text**: `<div>` below input, red for error (#ef4444), gray for helper (#6b7280), font-size 12px

    **Focus styles**: boxShadow `0 0 0 3px rgba(59,130,246,.2), inset 0 1px 0 rgba(255,255,255,0.9)`, borderColor `rgba(59,130,246,0.5)`, bg `rgba(255,255,255,0.85)`

    **Border styles**: normal = `1px solid rgba(0,0,0,0.1)`, error = `1px solid #ef4444`, bg = `rgba(255,255,255,0.72)`

    **Public API methods** (set on element instance):
    - `getValue()` — returns current value (handles contenteditable vs input)
    - `setValue(val)` — sets value, updates counter
    - `setError(msg)` — sets error text and red border (empty string clears)
    - `setHelper(msg)` — sets helper text
    - `setDisabled(bool)` — toggles disabled state
    - `checkValidity()` — validates required + maxLength, sets error if invalid, returns boolean

    **Events**: emit `input` on input event, emit `change` on change/blur — per D-02, D-03
