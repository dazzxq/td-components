    Create `src/feedback/td-tooltip.js` — port from DCMS `Tooltip` to `TdTooltip`.

    Per D-10: Auto-init global singleton. Importing this file auto-creates the tooltip instance and calls init(). Per architecture note, Claude decides whether to use custom element wrapper or pure JS class — use pure JS class (matching DCMS pattern, no need for `<td-tooltip>` tag since it's invisible infrastructure).

    **Class TdTooltip (instance-based, NOT static):**

    Constructor state:
    - `tooltip = null` — tooltip DOM element
    - `tooltipContent = null` — content container
    - `tooltipArrow = null` — arrow element
    - `currentElement = null`
    - `isVisible = false`
    - `hideTimeout = null`, `showTimeout = null`
    - `lastMouseEvent = null`
    - `_inited = false`
    - `_observer = null` (MutationObserver)

    Methods to port from DCMS:
    1. **init()** — Guard with _inited flag. Call createTooltipElement(), bindEvents(), preventNativeTooltipConflicts().
    2. **createTooltipElement()** — Create absolute-positioned div with: z-index 10010, hidden, pointer-events none, dark bg (rgba(30,30,30,0.75)), backdrop-filter blur(12px), white text, padding 8px 12px, rounded-[10px], max-width 280px. Create content container (relative z-1) and arrow element (8x8 rotated 45deg diamond). Append to body. Use `td-tooltip` class name.
    3. **bindEvents()** — Per D-12: Internal movement detection.
       - mouseenter (capture): Find closest [data-tooltip], clear hideTimeout, show tooltip
       - mouseleave (capture): Per D-12 critical fix — check if relatedTarget is still within same [data-tooltip] element (handles icon->text movement inside button). Only start hide timeout if truly leaving.
       - mousemove (capture): Secondary hover check for edge cases
       - scroll (capture): hide immediately
       - resize, blur: hide
       - Escape key: hide
    4. **show(element)** — Get content from data-tooltip. Handle custom colors per D-11: read `data-tooltip-color` for bg, compute accessible text color (simple approach: light text for dark bg, dark text for light bg). Set content, apply colors to tooltip + arrow. Position with rAF. Auto-hide after 30s.
    5. **hide()** — Clear timeouts, hide tooltip, reset currentElement.
    6. **position(element)** — Per D-11: Auto-positioning. Read `data-tooltip-position` attribute (top/bottom/left/right, default top). Calculate position based on element rect + scroll offset. Place arrow on opposite side. Clamp to viewport with 10px padding. Adjust arrow position when tooltip is clamped to edge.
    7. **preventNativeTooltipConflicts()** — Remove title attribute from elements with data-tooltip. Setup MutationObserver to do same for dynamically added elements.
    8. **disconnect()** — Disconnect MutationObserver.
    9. **isCurrentlyHovering()** / **isMouseOverElement(event, element)** — Helper methods for hover detection
    10. **getTooltipContent(element)** — Read data-tooltip attribute value. Skip DCMS permission check (not applicable to td-components).
    11. **_shadowRGBA(color, alpha)** / **_toComputedRGB(color)** — Color utility for shadow computation (port from DCMS).

    **Auto-init singleton (per D-10):**
    At bottom of file, after class definition:
    ```
    const tdTooltip = new TdTooltip();
    if (typeof document !== 'undefined') {
      if (document.readyState === 'loading') {
        document.addEventListener('DOMContentLoaded', () => tdTooltip.init());
      } else {
        tdTooltip.init();
      }
    }
    export { TdTooltip, tdTooltip };
    ```

    Export both the class (for testing/custom instances) and the singleton instance `tdTooltip`.
