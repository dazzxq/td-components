Fix by changing `afterRender()` click handler to update DOM classes directly instead of calling `setAttribute('checked')` / `removeAttribute('checked')` which triggers full re-render. BUT the `<button>` element has `disabled` attribute (line 218) which causes the browser to ignore cursor styles on the button itself. The `disabled` attribute on the native button already prevents click events, so `pointer-events-none` is redundant and harmful.

The thumb already has `top: 50%; transform: translateY(-50%)` via CSS, but the hover effect adds scale which makes it jump. The user wants this REMOVED entirely. Delete the mouseenter and mouseleave event listeners (lines 217-226) from `afterRender()`.

**Bug 5 - Dropdown bouncing:** The dropdown menu (`.td-dropdown-menu`) is positioned with `position: fixed` and updated via `_updatePosition()` in a RAF-throttled scroll handler. The bouncing is caused by the menu element having CSS transitions on it. The menu element at line 129 in `afterRender()` has no explicit transition, but the class includes `transition-all` from no explicit source. Check the `_menuElement.style.cssText` — there is no transition there. The bouncing likely comes from the scroll handler updating `top`/`left` on each RAF tick while CSS transitions smooth the movement, causing a rubber-band effect. Fix by adding `transition: none !important;` to the menu element's inline styles, or more precisely, ensure no transition is applied to positioning properties. Add to `_menuElement.style.cssText`: `transition: none;` to prevent any inherited or class-based transitions from affecting position updates during scroll.
